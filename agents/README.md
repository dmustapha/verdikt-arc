# Verdikt agents — autonomous payer + seller

Two real LLM agents that transact through Verdikt on Arc. Nothing here is a fixture: the **acceptance
criteria** and the **deliverable** are both generated by Claude at run time.

- **Payer agent** (`src/payer-agent.ts`) — given a plain-language goal, it reasons out machine-checkable
  acceptance criteria (a pytest, a JSON schema, or sources), funds the escrow on Arc, and signs a Task Offer.
- **Seller agent** (`src/seller-agent.ts`) — receives the offer + a brief, onboards onto Circle Gateway,
  generates the deliverable, and submits it for a verdict. Independent: it trusts the signed offer + the
  on-chain escrow and never sees the payer's tests.
- **Base** (`src/agent.ts`) — an agent is a wallet + a Verdikt client + an LLM brain playing a role. New
  roles (auditor, broker, route specialists) extend this; Verdikt never does the work, it settles between agents.

## Run
```bash
# 1. build the SDK the agents depend on (file:../sdk)
cd ../sdk && npm install && npm run build && cd ../agents

# 2. install + run (env supplies ANTHROPIC_API_KEY, DEMO_PAYER_KEY, WORKER_GATEWAY_KEY, ARC_RPC_URL)
npm install
WORKER_URL=https://verdikt-worker.fly.dev npx tsx --env-file=../.env src/run.ts          # all 3 outcomes
WORKER_URL=https://verdikt-worker.fly.dev npx tsx --env-file=../.env src/run.ts release   # one scenario
```

## What it proves (real Arc settlements)
| Scenario | Agents | Outcome |
|---|---|---|
| **Release** | diligent seller writes correct `build_sum` | tests pass → seller paid |
| **Refund** | hasty seller (terse brief) writes `sum/len`, misses the empty-list test | test fails → payer refunded |
| **Abstain** | seller answers a question the payer's sources don't cover | not grounded → payer refunded, fee 0 |

The refund scenario models the real gap between an informal brief and strict acceptance tests: the seller
builds to the terse ask, and the payer's hidden edge-case test still governs the money. The point is honest —
even a capable agent doesn't get paid for work that fails the payer's own definition of done.

## Notes
- Payer key escrows USDC (EIP-3009); seller key signs the artifact + pays the sub-cent fee from its Gateway
  balance (`seller.ensureOnboarded()` tops it up once, idempotently).
- Model: `REASONER_MODEL` (default `claude-sonnet-4-6`), the same family the verdict arbiter uses.
